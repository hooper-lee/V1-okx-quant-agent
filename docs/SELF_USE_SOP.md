# OKX Quant Agent 自用版 SOP

这份 SOP 不是开发文档，而是给你自己日常使用的操作说明。

适用场景：

- 系统已经部署在线上
- OKX、模型、新闻源已经基本可用
- 你当前主要是自己看盘、看策略、做验证
- 暂时不重点做域名、多人权限、正式运维

默认访问地址：

- 首页：`http://3.25.117.0:8010/`

## 先记住 4 个原则

1. 先看数据是否正常，再看策略建议。
2. 先跑自检，再决定要不要做回测或下单。
3. 没确认是模拟盘之前，不要随便点下单。
4. 每次大改配置后，都重新跑一次自检。

## 页面怎么理解

顶部菜单主要分成 8 个模块：

- `总览`：看当前策略、目标资金进度、账户实时和整体状态
- `策略`：切换当前策略、编辑参数、刷新分析
- `回测`：用当前策略做历史回测
- `账户`：看账户资产、持仓、账户原始返回
- `订单`：看挂单和历史订单
- `配置`：模型、OKX、自动化配置
- `数据源`：新闻源配置和抓取状态
- `自检`：统一检查各模块链路是否正常

## 每天的推荐使用顺序

### 1. 先看 `总览`

重点看这些位置：

- `System Online` 是否正常
- 右侧 `核心数据看板`
- `当前资金 / 目标资金 / 资金差额 / 周期剩余`
- `账户实时`
- `交易记录墙`

你主要看三件事：

- 当前策略是不是你想看的那一套
- 账户资产是不是正常显示
- 目标资金和当前资金有没有明显异常

如果这里已经不正常，不要直接去下单，先去 `账户` 或 `自检`。

### 2. 再看 `账户`

点击 `账户 -> 刷新账户`

你要确认：

- `source` 是不是你预期的来源
- `mode` 是 `paper` 还是 `live`
- `total_equity` 有没有正常返回
- 资产表里有没有你账户里的币种

判断建议：

- 如果你只是观察系统，优先用 `paper`
- 如果你现在是实盘账户，必须确认 `mode=live`
- 如果返回 `demo-fallback`，说明真实账户链路没打通，先不要继续交易

### 3. 然后跑一次 `自检`

点击：

- `自检 -> 运行自检`

先看左侧状态卡片，再看右侧详细输出。

重点关注：

- `LLM`
- `新闻源`
- `看板快照`
- `回测`
- `OKX`
- `Prompt 预览`

怎么判断：

- `正常`：可以继续
- `fallback`：还能用，但有降级
- `失败`：先修，不要做后续关键操作

如果 `OKX` 失败，优先停下来处理网络、鉴权或模式问题。

## 想看策略时怎么操作

### 1. 进入 `策略`

你会用到这些按钮：

- `刷新分析`
- `新增策略`
- `编辑当前策略`

先做这个动作：

1. 选中一个当前策略
2. 点 `刷新分析`

你要看的核心信息：

- 当前交易对
- 当前周期
- 信号是 `buy / sell / hold`
- Agent 决策
- 风控结果
- 技术指标面板

### 2. 如果要改策略参数

点击：

- `编辑当前策略`

最常改的是：

- `symbol`
- `timeframe`
- `target_capital`
- `target_horizon_days`
- `leverage`
- `margin_mode`
- `take_profit_pct`
- `stop_loss_pct`
- `entry_allocation_pct`
- `max_position_pct`

参数修改建议：

- 一次只改 1 到 3 个核心参数
- 改完后立刻 `保存当前策略`
- 然后重新 `刷新分析`
- 再去 `回测`

## 想验证策略时怎么操作

### 1. 去 `回测`

操作顺序：

1. 选择策略
2. 确认交易对
3. 确认周期
4. 确认初始资金
5. 点 `运行回测`

重点看：

- 收益率
- 最大回撤
- 交易次数
- 权益曲线
- 最近成交记录

建议：

- 不要只看收益率
- 一定同时看最大回撤和交易次数
- 如果回测结果明显离谱，再回头改策略参数

### 2. 回测结果值得保存时

点击：

- `保存本次回测`

这样后面可以做历史比较。

## 想看新闻时怎么操作

进入：

- `数据源`

你会用到：

- `保存新闻源配置`
- `刷新新闻`

你主要看：

- 每个新闻源是不是抓到了内容
- 当前状态是不是成功
- 输出区域里有没有最新内容

如果新闻源挂了：

- 先看是不是 RSS 地址失效
- 再看是不是目标站点暂时不可访问

## `配置` 页面该怎么用

这个页面现在主要分 3 类：

- 模型配置
- OKX 配置
- 自动化配置

### 1. 模型配置

常用字段：

- `OPENAI_API_KEY`
- `LLM_MODEL`
- `LLM_TEMPERATURE`
- `OPENAI_BASE_URL`

操作方式：

1. 改配置
2. 点 `保存模型配置`
3. 点 `测试模型连接`

只有测试通过，才建议继续依赖 AI 输出。

### 2. OKX 配置

常用字段：

- `OKX_API_KEY`
- `OKX_API_SECRET`
- `OKX_PASSPHRASE`
- `OKX_REST_BASE`
- `OKX_USE_PAPER`
- `OKX_ADAPTER`

当前线上建议：

- `OKX_ADAPTER`：优先用 `ccxt`
- `OKX_USE_PAPER`：
  - 只观察或联调时用 `true`
  - 实盘确认后再用 `false`

操作方式：

1. 改配置
2. 点 `保存 OKX 配置`
3. 先测 `公共行情`
4. 再测 `私有账户`
5. 再看 `执行诊断`

### 3. 页面访问密码

现在这个密码的作用不是整个系统首页门禁。

它当前只用于保护：

- 配置读取
- 配置保存
- 配置相关接口

所以现在的行为是：

- 你可以直接看首页、账户、策略、回测
- 但进入配置相关敏感操作时，才会要求密码

### 4. 自动化配置

如果你现在是自用，建议先保持：

- `auto_trade_enabled=false`

先把系统看稳，再考虑自动化。

## 什么情况下可以点“模拟下单”

只有同时满足这些条件才建议尝试：

1. `账户` 页正常
2. `自检` 页 OKX 正常或至少不失败
3. 当前策略已经刷新分析
4. 你知道当前是 `paper` 还是 `live`
5. 你确认这次下单是测试，不是误触

如果你不确定，就先不要点。

## 日常最小操作流程

每天最简流程可以按这个顺序：

1. 打开首页
2. 看 `总览`
3. 进 `账户` 点 `刷新账户`
4. 进 `自检` 点 `运行自检`
5. 进 `策略` 点 `刷新分析`
6. 有需要再去 `回测`
7. 确认一切都正常后，再决定是否手动下单

## 看到这些情况时怎么处理

### 情况 1：账户显示 `demo-fallback`

说明真实账户链路没打通。

先检查：

- `OKX_USE_PAPER`
- API Key / Secret / Passphrase
- `OKX_ADAPTER`
- OKX 网络诊断

### 情况 2：OKX 公共行情失败

先不要怀疑策略。

优先检查：

- 当前网络
- `OKX_REST_BASE`
- `OKX_ADAPTER`
- 服务器当前出口是不是被 OKX 限制

### 情况 3：自检页面失败很多

处理顺序建议：

1. 先修 `OKX`
2. 再修 `LLM`
3. 再修 `新闻源`
4. 最后再看策略和回测

### 情况 4：分析超时

优先检查：

- 模型接口是否稳定
- `LLM_MODEL` 是否可用
- `OPENAI_BASE_URL` 是否正确

如果只是偶发超时，先重试一次。

## 当前这套自用模式的建议边界

现阶段更适合：

- 看盘
- 看账户
- 刷策略分析
- 跑回测
- 小心做手动验证

现阶段不建议一上来就做：

- 全自动实盘
- 高频频繁下单
- 一边改配置一边直接实盘联动

## 建议你接下来固定下来的 3 件事

1. 固定一套主策略
2. 固定当前运行模式是 `paper` 还是 `live`
3. 固定每天先看 `账户 + 自检 + 策略`

这样你后面用起来会稳定很多。
